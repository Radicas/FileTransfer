# FileTransfer 项目上下文

## 项目概览

**FileTransfer** 是一个跨平台的局域网文件传输系统，基于 C++17 和 Qt6 开发。项目采用分层架构设计，支持 Windows 和 macOS 系统，无需复杂配置即可自动发现局域网中的其他设备并进行文件传输。

### 核心特性

- **自动设备发现**：UDP广播自动发现局域网设备，心跳检测保持连接
- **双面板设计**：左侧本地文件，右侧远程文件，直观对比操作
- **拖拽传输**：支持拖拽文件进行上传/下载，操作简单直观
- **实时进度**：显示传输进度、速度和剩余时间
- **多主题支持**：内置多种UI主题（Modern、Cyberpunk、Industrial）
- **跨平台兼容**：支持 Windows（MSVC/MinGW）和 macOS（Clang）

## 项目架构

项目采用**分层架构**设计，遵循 SOLID 原则，模块化程度高，易于扩展。

```
src/
├── app/                    # 应用层 - 程序入口
│   └── main.cpp           # 主函数，初始化各模块
│
├── common/                 # 公共层 - 通用工具和基础设施
│   ├── logger/            # 日志系统（跨平台彩色输出）
│   ├── theme/             # 主题管理（QSS样式）
│   ├── messagebus/        # 消息总线（发布-订阅模式）
│   ├── json/              # JSON工具类
│   └ utils/               # 工具类（防火墙、版本管理）
│
├── core/                   # 核心层 - 业务逻辑
│   ├── network/           # 网络通信模块
│   │   ├── networkprotocol.h    # 通信协议定义
│   │   ├── devicediscoverer     # UDP设备发现服务
│   │   └ filetransferservice    # TCP文件传输服务
│   ├── device/            # 设备管理模块
│   │   └ devicemanager          # 设备管理器（单例）
│   └ filesystem/          # 文件系统模块
│   │   └ filesystemmodel        # 文件列表模型
│
└── ui/                     # 界面层 - 用户交互
    ├── mainwindow/        # 主窗口
    └ widgets/             # 自定义控件
        ├── devicetreeview         # 设备树视图
        ├── filebrowserpanel       # 文件浏览器面板
        └ transferprogresspanel    # 传输进度面板
```

## 核心模块详解

### 1. 网络模块 (core/network)

#### networkprotocol.h - 网络协议定义

定义了所有网络通信的消息类型和数据结构：

**消息类型枚举**：
- `DiscoverBroadcast` - 设备发现广播
- `DiscoverResponse` - 设备发现响应
- `DiscoverHeartbeat` - 心跳消息
- `DiscoverLeave` - 设备离线通知
- `FileListRequest/Response` - 文件列表请求/响应
- `FileDownloadRequest/Response` - 文件下载请求/响应
- `FileUploadRequest/Response` - 文件上传请求/响应
- `TransferData` - 文件数据块
- `TransferComplete/Error/Cancel` - 传输状态消息

**数据结构**：
- `DeviceInfoData` - 设备信息（ID、名称、IP、端口、操作系统、用户名）
- `FileInfoData` - 文件信息（名称、路径、类型、大小、修改时间、权限）
- `NetworkMessage` - 网络消息封装类（序列化/反序列化）

#### devicediscoverer.h - 设备发现服务

**单例模式**，负责局域网设备发现：

- 通过 UDP 广播发现设备
- 定时发送心跳消息（5秒间隔）
- 检测设备超时离线（15秒超时）
- 维护在线设备列表

**关键接口**：
```cpp
static DeviceDiscoverer& instance();
bool initialize(const DeviceInfoData& localDeviceInfo);
void start();
void stop();
QList<DeviceInfoData> getOnlineDevices() const;
```

**信号**：
- `deviceDiscovered(DeviceInfoData)` - 发现新设备
- `deviceOffline(QString)` - 设备离线
- `deviceUpdated(DeviceInfoData)` - 设备信息更新

#### filetransferservice.h - 文件传输服务

**单例模式**，基于 TCP 的文件传输：

- 监听 TCP 端口接收传输请求
- 支持文件列表查询、下载、上传
- 分块传输（64KB每块）
- 传输进度报告

**关键接口**：
```cpp
static FileTransferService& instance();
bool initialize();
void start();
QString requestFileList(const QString& deviceId, const QString& path);
QString requestDownload(const QString& deviceId, const QString& remotePath, const QString& localPath);
QString requestUpload(const QString& deviceId, const QString& localPath, const QString& remotePath);
void cancelTransfer(const QString& taskId);
```

**传输状态**：
- `Pending` - 等待中
- `Preparing` - 准备中
- `Running` - 进行中
- `Completed` - 已完成
- `Failed` - 已失败
- `Cancelled` - 已取消

### 2. 设备管理模块 (core/device)

#### devicemanager.h - 设备管理器

**单例模式**，管理所有发现的设备：

- 维护设备列表模型（`DeviceListModel`）
- 跟踪设备状态（在线/离线/繁忙/错误）
- 统计传输数据量
- 与设备发现服务集成

**设备状态枚举**：
```cpp
enum class DeviceStatus {
    Offline,    // 离线
    Online,     // 在线
    Busy,       // 繁忙（正在传输）
    Error       // 错误状态
};
```

### 3. 文件系统模块 (core/filesystem)

#### filesystemmodel.h - 文件系统模型

提供文件列表数据模型：

- `FileSystemType` - 本地/远程文件系统类型
- `FileItem` - 文件项数据结构
- `FileListModel` - Qt模型类，用于视图显示

### 4. 公共模块 (common)

#### logger.h - 日志系统

**单例模式**，跨平台彩色日志输出：

- 日志级别：Debug（青色）、Info（绿色）、Warning（黄色）、Error（红色）
- 支持控制台和文件输出
- 线程安全
- Windows/Linux/macOS 兼容

#### messagebus.h - 消息总线

**单例模式**，发布-订阅模式实现模块间解耦通信：

- 类型安全的消息传递
- 支持同步和异步消息处理
- 支持消息优先级
- 线程安全
- 支持请求-响应模式

**消息类型**（messagetypes.h）：
- `DataAdded` - 数据添加
- `DataUpdated` - 数据更新
- `DataDeleted` - 数据删除
- 等...

#### theme/thememanager.h - 主题管理

**单例模式**，管理 UI 主题：

- 内置主题：Modern、Cyberpunk、Industrial
- QSS 样式文件加载
- 动态主题切换

#### utils/firewallmanager.h - 防火墙管理

管理防火墙端口规则：

- Windows 防火墙规则添加/删除
- macOS 防火墙配置（开发中）

---

### 5. 界面模块 (ui)

#### mainwindow.h - 主窗口

应用程序主界面框架：

- **左侧**：设备树视图（`DeviceTreeView`）
- **中间**：双文件浏览器面板（本地 + 远程）
- **底部**：传输进度面板（`TransferProgressPanel`）
- 菜单栏、工具栏、状态栏

**初始化流程**：
```cpp
// 1. 初始化主题
ThemeManager::instance().init();
ThemeManager::instance().applyTheme(&app);

// 2. 初始化日志
Logger::instance().info("应用程序启动");

// 3. 初始化防火墙
FirewallManager::instance().initialize(nullptr, false);

// 4. 创建主窗口
MainWindow window;
window.show();
```

#### widgets/filebrowserpanel.h - 文件浏览器面板

文件浏览和拖拽操作：

- 本地/远程文件系统浏览
- 路径导航（上级、根目录、指定路径）
- 拖拽支持（发起拖拽、接收拖拽）
- 右键菜单（复制、粘贴、删除、新建文件夹）

#### widgets/devicetreeview.h - 设备树视图

显示在线设备列表：

- 本机设备节点
- 远程设备节点
- 设备状态图标

#### widgets/transferprogresspanel.h - 传输进度面板

显示传输任务列表：

- 任务信息（文件名、进度、大小、速度、状态）
- 取消传输按钮
- 清除已完成任务

## 构建系统

### CMake 配置

项目使用现代化的 CMake 配置：

```cmake
cmake_minimum_required(VERSION 3.16)
project(FileTransfer VERSION 1.0.0 LANGUAGES CXX)

set(CMAKE_CXX_STANDARD 17)
set(CMAKE_AUTOMOC ON)
set(CMAKE_AUTORCC ON)
set(CMAKE_AUTOUIC ON)

find_package(Qt6 REQUIRED COMPONENTS Core Widgets Sql Network Concurrent)
```

### 编译器选项

- **MSVC**: `/utf-8 /W4`
- **GCC/Clang**: `-Wall -Wextra -Wpedantic`

### 构建脚本

项目提供 Python 构建脚本：

```bash
python scripts/build.py -c msvc -b Release   # Windows MSVC
python scripts/build.py -c mingw -b Release  # Windows MinGW
python scripts/build.py -c clang -b Release  # macOS
```

## 总结

FileTransfer 是一个架构清晰、模块化程度高的跨平台文件传输应用。项目采用分层设计，将网络通信、设备管理、文件系统和用户界面分离，通过消息总线实现模块间解耦通信。代码遵循 Google C++ Style Guide，使用 Doxygen 风格中文注释，具有良好的可维护性和可扩展性。
